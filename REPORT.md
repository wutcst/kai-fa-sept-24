# 软件工程实践二小组实训报告

## 项目概述

本项目基于 `world-of-zuul` 文本冒险游戏进行扩展，目标是完成一个具有完整交互流程和基础工程化能力的 Maven 项目。玩家可以在校园房间中移动，查看房间信息，管理物品，并通过特殊物品改变自身能力。

## 需求完成情况

已完成的结构优化和功能扩充包括：

1. 房间支持多个物品，物品包含名称、描述和重量。
2. `look` 命令可以显示当前房间、出口和物品清单。
3. `back` 命令可以连续回退到之前经过的房间。
4. 新增 `Player` 类，保存玩家基本信息、当前位置、背包和最大负重。
5. 新增 `take`、`drop`、`drop all`、`items` 命令，支持物品拾取、丢弃和查看。
6. 新增 `eat cookie` 命令，玩家吃掉魔法饼干后可以提高负重上限。
7. 新增传送房间，玩家进入后会被随机传送到其他房间。
8. 使用命令注册表组织命令处理逻辑，降低新增命令时对主流程的影响。
9. 新增钥匙和锁门机制，玩家需要先找到钥匙才能进入资料室。
10. 新增通关目标，玩家取得宝藏并回到入口后游戏胜利。
11. 新增分数系统，拾取物品、解锁房间和通关都会增加分数。
12. 新增生命值系统，危险房间会扣除生命值，生命值归零后游戏失败。
13. 新增任务、状态和地图命令，玩家可以查看当前目标、生命分数、背包和探索记录。
14. 新增存档和读档命令，可以保存和恢复当前游戏状态。
15. 新增命令别名和方向简写，提升终端交互体验。

## 设计说明

项目采用简单的面向对象结构：

- `Game` 负责创建游戏世界、注册命令和驱动主流程。
- `Room` 负责保存房间描述、出口和房间物品。
- `Player` 负责保存玩家当前位置、背包和负重能力。
- `Item` 负责描述游戏物品。
- `Parser`、`Command`、`CommandWords` 负责命令解析和命令合法性判断。

命令处理逻辑通过 `Map<String, CommandHandler>` 注册。这样新增命令时，只需要新增处理方法并注册命令，不需要不断扩大 `processCommand` 中的条件判断。

游戏目标设计为“找到钥匙、进入资料室、取得宝藏、回到入口”。这一目标把物品系统、地图移动、锁门机制和胜利条件串成一条完整流程，便于演示和答辩。

## 工程化说明

项目已改为标准 Maven 结构：

- 主代码位于 `src/main/java`。
- 测试代码位于 `src/test/java`。
- 使用 JUnit 5 编写自动化测试。
- `mvn package` 可以生成可执行 jar。
- GitHub Actions 会在 push 和 pull request 时自动运行测试和打包。

## 测试说明

自动化测试覆盖以下场景：

- 未知命令、`help`、`quit`
- 正常移动和无效方向
- `look` 房间和物品展示
- 连续 `back`
- `take`、`drop`、`drop all`、`items`
- 超重物品不能拾取
- `eat cookie` 后负重提升
- 进入传送房间后到达合法房间
- 方向别名和 `get` 命令别名
- 钥匙锁门和解锁
- 拿到宝藏后回到入口通关
- 危险房间扣除生命值
- `quest`、`score`、`status`、`map`
- `save` 和 `load`

本地验证命令：

```bash
mvn test
mvn package
java -jar target/zuul-1.0-SNAPSHOT.jar
```
